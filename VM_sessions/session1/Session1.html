Overview

In computing, a shell is a computer programme that allows you to communicate with Unix operating systems (Linux and MacOS). We do this using a command-line interface where we input lines of text that instruct the shell on what to do. The most popular Unix shell is Bash (Bourne Again SHell), which is the default on most modern unix systems. This is what we will be using to perform our tasks, via the Terminal application.

Typically, we use graphical interfaces to use a computer (clicking a mouse and using menus). However, some tasks can take hours to do using this method, whereas using a command line could take only seconds. Additionally, we can create scripts to automate tasks, meaning we can do other things while the shell is working.

The Unix shell is a powerful tool, with a high action-to-keystroke ratio, making complex and powerful tasks easier. It is essential for accessing (and managing) networked machines and high-performance computers.

Using the shell can seem daunting at the beginning, but only a few "words" (commands) are needed to perform a wide variety of tasks. Unlike when normally using a computer, there are no choices automatically presented and so it is necessary to learn some essential commands. As with most computing problems, using the programme documentation or a simple Google search can usually tell you what command and options are needed to do whatever you require.

A Typical Problem

Iris Spectra, a chemistry student, has been doing a research project involving IR spectroscopy. They have performed 100 synthesis experiments and collected the spectra as individual text files. Each file needs processed to normalise the data, get the peak positions and assign the peaks to functional groups. They also want to generate a summary of the 100 experiments to present to their supervisor. In addition to this huge task, the results need written up in 2 weeks so that the paper can appear in a special additional of The Journal of Immaterial Science.

To speed up the process, Iris has created a programme called IR_spectra.sh to perform the analysis, but if it was run by hand using a graphical interface they will have to select and open all 100 files. If IR_spectra.sh takes 2 minutes to run, the whole process will over 3 hours. Using the shell, Iris can instead assign the task to their computer while focussing on writing the paper.

In this session will explore the ways Iris can achieve this. More specifically, the practical explains how they can use a command shell to run the IR_spectra.sh program, using loops to automate the repetitive steps of entering file names, so that Iris’ computer can work while they write the paper.

As a bonus, once Iris has put a processing pipeline together, they will be able to use it again whenever they collect more data.

In order to achieve the task, Iris needs to know how to:

  • Navigate to a file/directory
  • Create a file/directory
  • Chain commands together
  • Retrieve a set of files
  • Iterate over files
  • Run a shell script containing the pipeline
